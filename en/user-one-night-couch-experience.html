Creation of idea and UX/UI for the app from scratch

*in progress

I am developing an app from scratch as a UX writer, project lead, and creative director. The app’s concept is low-cost one-night stays in New York City within a network of friends or friends of friends.

At this stage, I have:

✅ created the core concept of the service

✅ developed its name

✅ conducted user research

✅ onboarded a UI/UX Designer to collaborate on the project

✅ created the information architecture

✅ written UX copy for mid-fidelity wireframes, including microcopy for headlines, subheadlines, CTAs, pop-ups, and buttons

The project is ongoing, so you can expect more design iterations soon.

In the mid-fidelity prototype we mapped out the app flow for the first 3 must-have features from the Feature Roadmap: account creation, the "Couch" ad creation with AI feature for Persona 1 (Emily), and the Booking for Persona 2 (Ethan).

Account creation

Ad creation flow we started with uploading photos, as this is the key AI feature at this stage. Once the photos are uploaded, the AI assists in filling out the input fields, which users can edit if necessary.

Ad creation, p. 1

After the AI assists in filling out some input fields, the user must manually add the remaining information and set personal preferences to create the house rules. Once everything is complete, users can preview or post the ad immediately.

Ad creation, p. 2

Once the user selects an ad and presses the 'Book' button, they have the primary option to book a stay for tonight or the secondary option to book for a later date. Before sending the inquiry, the user can leave a note for the host regarding their stay.
